# 화면 및 UI 설계

## 개요

본 문서는 이슈 `#2 [FEAT] 화면 및 UI 설계`의 작업 내용을 정리한다.

서비스는 시각장애 아동을 위한 AI 그림책 독서 보조 서비스이므로, 화면 구조는 단순하고 버튼은 크게 배치하며, 상태 변화는 색상과 텍스트를 함께 제공하는 방향으로 설계한다.

디자인 시안은 Figma `OSS design`에서 작업한 내용을 기준으로 하며, 현재 구현된 CSS와 화면 구성에 맞춰 정리한다.

참고 디자인:

- [OSS design](https://www.figma.com/design/lBsjIdujvO0PJeFmzYBxoc/OSS_design?node-id=2001-212&t=O53mbnpXJ1ECK79p-1)

## 디자인 목표

- 시각장애 아동과 보호자가 쉽게 이해할 수 있는 직관적인 화면 구성
- 독서 시작, 카메라 시작, 음성 안내 등 주요 행동을 명확하게 강조
- 웹캠 권한, 객체 탐지, 손끝 추적, TTS 안내 상태를 한눈에 확인
- 모바일과 데스크톱 환경에서 모두 사용할 수 있는 반응형 레이아웃
- 색상만으로 상태를 구분하지 않고 아이콘, 문구, 버튼 상태를 함께 제공

## 공통 UI 스타일

### 색상

현재 전역 CSS는 `src/styles/index.css`의 CSS 변수로 색상을 관리한다.

```css
--color-primary: #7a9e7e;
--color-secondary: #f4d06f;
--color-accent: #e07a5f;
--color-surface: #f8f5ef;
--color-text: #3a3a3a;
--color-sub-text: #767676;
```

### 컴포넌트 스타일

- 버튼은 최소 높이를 크게 설정하여 터치하기 쉽게 설계한다.
- 주요 CTA 버튼은 `primary`, 보조 상태는 `secondary`, 오류나 종료 동작은 `accent` 색상을 사용한다.
- 카드 컴포넌트는 기능 소개 영역에서 동일한 구조로 반복 사용한다.
- 포커스 상태는 `focus-visible` outline을 제공하여 키보드 접근성을 고려한다.

### 반응형 기준

- `1180px` 이하: 홈 화면 히어로 영역을 1열 구조로 변경하고 기능 카드를 2열로 배치
- `760px` 이하: 모바일 기준으로 버튼을 전체 너비에 가깝게 확장하고 기능 카드를 1열로 배치
- 웹캠 화면은 작은 화면에서 진행 단계를 2열로 재배치하고 제어 패널을 하단으로 내린다.

## 화면 구성

### 1. 메인 화면

경로:

```text
/
```

역할:

- 서비스 첫 진입 화면
- 서비스 목적과 핵심 기능을 소개
- 독서 화면으로 진입하는 주요 CTA 제공

주요 구성:

- 상단 로고 영역
- 히어로 문구: `손끝으로 읽는 AI 그림책`
- 서비스 설명 문구
- `독서 시작하기` 버튼
- 주요 기능 카드 3개

주요 기능 카드:

- 그림책 탐색
- 손끝 인식
- 음성 안내

와이어프레임:

```text
┌──────────────────────────────────────────────┐
│ Logo Fingertips                              │
├──────────────────────────────────────────────┤
│                                              │
│  손끝으로 읽는                               │
│  AI 그림책                         [이미지]  │
│                                              │
│  시각장애 아동을 위한 독서 보조 서비스       │
│  [독서 시작하기]                             │
│                                              │
├──────────────────────────────────────────────┤
│ 주요 기능                                    │
│ With us, the world is at my fingertips.      │
│                                              │
│ [그림책 탐색] [손끝 인식] [음성 안내]        │
└──────────────────────────────────────────────┘
```

접근성 고려:

- CTA 버튼 크기를 크게 설정하여 보호자와 아동 모두 쉽게 누를 수 있도록 한다.
- 핵심 문구는 짧고 명확하게 유지한다.
- 기능 카드는 아이콘, 제목, 설명을 함께 제공한다.

### 2. 웹캠 독서 화면

경로:

```text
/reading
```

역할:

- 실시간 카메라 영상을 통해 그림책과 교구를 인식하는 핵심 화면
- 웹캠 실행 상태와 AI 처리 단계를 안내
- 음성 안내를 다시 들을 수 있는 하단 TTS 영역 제공

주요 구성:

- 상단 로고 영역
- 진행 상태 표시 영역
- 웹캠 프레임
- 카메라 제어 패널
- 하단 TTS 안내 바

진행 상태:

```text
웹캠 인식 → 객체 탐지 → 손끝 추적 → TTS 음성 안내
```

상태 표현:

- 성공: 초록색, 체크 아이콘
- 대기/주의: 노란색, 알림 아이콘
- 실패: 주황색, 닫기 아이콘

와이어프레임:

```text
┌────────────────────────────────────────────────────┐
│ Logo Fingertips                                    │
├────────────────────────────────────────────────────┤
│  ● 웹캠 인식 ─ ● 객체 탐지 ─ ● 손끝 추적 ─ ● TTS  │
│                                                    │
│ ┌──────────────────────────────┐ ┌───────────────┐ │
│ │                              │ │ 카메라 상태   │ │
│ │        실시간 웹캠 화면       │ │               │ │
│ │                              │ │ [카메라 시작] │ │
│ │                              │ │ [카메라 종료] │ │
│ └──────────────────────────────┘ └───────────────┘ │
├────────────────────────────────────────────────────┤
│ [캐릭터] 웹캠으로 책과 교구를 비추면 AI가 안내해요 │
│                                      [설명 다시 듣기]│
└────────────────────────────────────────────────────┘
```

접근성 고려:

- 카메라가 실행되지 않은 상태에서도 안내 문구를 보여준다.
- 권한 거부, 브라우저 미지원, 카메라 오류 상태를 각각 문구로 안내한다.
- 버튼은 `카메라 시작`, `카메라 종료`, `설명 다시 듣기`처럼 동작 중심 문구를 사용한다.
- 진행 상태는 색상뿐 아니라 아이콘과 텍스트를 함께 사용한다.
- 웹캠 화면 제목은 시각적으로 숨겨진 `sr-only` 텍스트로 제공한다.

### 3. 사용 가이드 화면

경로:

```text
/guide
```

역할:

- 서비스 사용 방법을 단계별로 설명
- 독서 시작, 음성 안내, 페이지 넘기기, 북마크 기능 안내

주요 구성:

- 뒤로가기 버튼
- 가이드 제목
- 단계별 사용 설명

## UI 검토 결과

### 완료된 항목

- 메인 화면에서 서비스 목적과 독서 시작 동선을 명확하게 제공
- 웹캠 화면에서 실시간 카메라 영역과 제어 버튼을 분리
- AI 처리 과정을 진행 상태 UI로 표현
- TTS 안내 영역을 하단 고정형 바 형태로 구성
- 모바일 화면에서 버튼과 카드를 1열로 배치하여 터치 사용성 확보
- 카메라 권한 및 오류 상태에 대한 안내 문구 제공

### 개선 예정 항목

- 실제 객체 탐지 결과가 연결되면 감지 박스와 라벨 스타일을 검토한다.
- 손끝 추적 결과를 웹캠 프레임 위에 표시하는 UI를 추가한다.
- TTS 재생 중, 일시정지, 다시 듣기 상태를 버튼 상태로 세분화한다.
- 가이드 화면의 시각 스타일을 홈/독서 화면과 통일한다.
- 스크린 리더 사용 흐름을 기준으로 문구와 aria 속성을 추가 검토한다.
